# Security

If you discover a vulnerability affecting Mocha Messenger, Mocha Passport, the Notification Server, or Switchboard, please do **not** open a public issue with sensitive details.

Use a private contact path chosen by the Mocha team for responsible disclosure.

Until a dedicated security mailbox is published, avoid posting:

- authentication bypass details
- token or session handling flaws
- admin or moderation vulnerabilities
- infrastructure or deployment weaknesses
- anything that could be used to attack the public service

Public issues are fine for:

- general hardening suggestions
- non-sensitive compatibility concerns
- documentation clarifications
