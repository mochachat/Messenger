# GitHub Discussions Plan

GitHub Discussions would be a strong fit for the public Messenger repository.

## Why Discussions make sense here

- they separate brainstorming from bug tracking
- they give add-ins and UI ideas a better home than raw issues
- they make it easier to gather compatibility feedback without implying every
  thread is a direct product commitment

## Suggested categories

- `Ideas`
- `UI Feedback`
- `Compatibility`
- `Add-ins`
- `Questions`
- `Show and Tell`

## Suggested use

- use issues for concrete bugs and tightly scoped feature requests
- use discussions for broader design conversation, add-ins planning, and
  exploratory community ideas

## Maintainer note

When a discussion turns into a concrete action item, open or link a public
 issue and continue from there.
