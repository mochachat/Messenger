# Add-ins

Mocha Messenger is expected to have room for future add-ins or companion
integrations, but this is still a forward-looking area.

## What this means today

- add-ins are not a broad public end-user platform today
- public feedback about what an add-in system should do is still useful
- the Mocha team wants the eventual model to be useful, understandable, and
  safe

## Good public feedback topics

- what kinds of add-ins would be worth having
- how visible permissions and trust prompts should be
- whether add-ins should focus on communication features, service
  integrations, utility tools, or all three
- what an add-in hub should look like for end users

## Public design goals

If Mocha exposes add-ins later, strong goals would likely include:

- clear trust boundaries
- visible install and removal flows
- compatibility with Messenger's core user experience
- room for community ideas without weakening the service model
