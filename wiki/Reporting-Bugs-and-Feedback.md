# Reporting Bugs and Feedback

Public issues and discussions are useful when they are specific and reproducible.

## Include these details when possible

- build number
- Windows version
- whether custom themes or shell modifications are in use
- what you were doing when the issue happened
- whether it happens every time
- screenshots or short videos if available

## Good feature requests explain

- the user problem
- the expected behavior
- whether there is classic Messenger precedent
- why the idea belongs in Mocha

## Status and outages

If something looks like a service outage, check the public status page first:

- [https://mochachat.net/support/status.html](https://mochachat.net/support/status.html)
